`webstable32-clean-stable-build`

v32 принята как рабочая стабильная сборка.

## 2. Следующий этап

Следующий этап: `v33-artistic-recipes-upgrade`.

Цель: доработать рецепты так, чтобы они стали художественными, живыми, уютными и логичными, а не шаблонными.

## 3. Правило по изображениям

Картинки нужно добавить, но НЕ генерировать автоматически.

Запрещено:
- запускать AI image generation без прямого разрешения Лизы;
- считать фразу «давай картинки» разрешением на генерацию;
- генерировать изображения для блюд, логотипа, иконки или персонажа без отдельного подтверждения.

Разрешено:
- искать реальные изображения в открытых источниках;
- подбирать stock/photo candidates;
- скачивать и подключать стабильные локальные файлы, если права и источник подходят;
- составлять таблицу фото-кандидатов;
- отмечать рецепты как `needs_generation`, если реального фото нет.

AI-генерация возможна только после отдельной фразы Лизы в явном виде, например:

> можно генерировать эти фото

## 4. Почему прошлый подход с фото не подошёл

Прямые внешние ссылки на stock-фото дали проблемы:
- чёрные карточки;
- нестабильную загрузку;
- картинки появлялись только после открытия рецепта;
- часть ссылок не была прямыми image assets.

Для следующего фото-этапа использовать другой подход:

1. Не вставлять страницы Unsplash/Pexels/Pixabay как `src`.
2. Не полагаться на нестабильные remote hotlinks.
3. Сначала собрать таблицу кандидатов.
4. Потом использовать стабильные локальные assets в папке проекта, например:
   - `assets/food/syrniki.jpg`
   - `assets/food/chicken-soup.jpg`
   - `assets/food/mashed-potatoes-cutlets.jpg`
5. Подключать фото через отдельный слой/JSON после проверки.

## 5. Художественная доработка рецептов

Каждый рецепт должен стать маленькой карточкой заботы.

Для каждого рецепта доработать:

- название;
- короткое описание;
- `Совет от Лисы`;
- `Хитрость дня`;
- `Милое замечание`;
- шаги готовки;
- финальный комментарий;
- таймеры, если они нужны;
- логичность огня и времени;
- отсутствие повторов;
- отсутствие тяжёлых простыней текста.

## 6. Тон рецептов

Тон:
- уютный;
- живой;
- с юмором;
- без кулинарного снобизма;
- без перегруза;
- дружелюбный;
- по делу;
- немного «Лиса рядом на кухне».

Не писать:
- слишком длинные абзацы;
- одинаковые шаблонные советы во всех рецептах;
- повторы названия блюда в каждом блоке;
- «ошибочка» в каждом шаге;
- бессмысленные таймеры;
- фразы ради фраз.

## 8. Формат работы по v33

Сначала подготовить тексты на согласование, без немедленного внедрения в код.

Для каждого рецепта показать:

- обновлённое название;
- короткое описание;
- совет от Лисы;
- милое замечание;
- шаги в структуре v31;
