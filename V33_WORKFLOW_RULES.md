# Cozy Foodie — V33 Workflow Rules

## 1. Основное правило работы

Лиза утвердила стиль художественной доработки рецептов на примере сырников.

Дальше не спрашивать подтверждение по каждому рецепту.

Ассистент должен работать самостоятельно партиями:
- переписывать рецепты художественно;
- делать тексты уютными, живыми и логичными;
- сохранять структуру v31/v32;
- готовить фото-план без генерации;
- показывать результат уже готовыми партиями.

## 2. Когда подтверждение Лизы НЕ нужно

Подтверждение не нужно для:
- художественной переписки рецептов в утверждённом стиле;
- подготовки первой партии рецептов;
- подготовки фото-плана;
- поиска реальных фото-кандидатов в открытых источниках;
- создания markdown-документов с планами, правилами, текстами;
- формулировок советов, милых замечаний, финальных комментариев;
- мелкой текстовой чистки без изменения логики приложения.

## 3. Когда подтверждение Лизы нужно обязательно

Подтверждение нужно только в крайних случаях:

1. AI-генерация изображений.
   - Нельзя генерировать изображения без прямой фразы Лизы.
   - Фразы вроде «давай картинки» или «переходим к фото» не считаются разрешением.

2. Внедрение крупного кода в приложение.
   - Особенно если меняется стабильная сборка `webstable32-clean-stable-build`.

3. Архитектурные изменения.
   - Авторизация.
   - Аккаунты пользователей.
   - Backend.
   - База данных.
   - Supabase/Firebase.

4. Необратимые решения.
   - Удаление файлов.
   - Переписывание основной структуры.
   - Замена большого объёма данных без backup.

5. Выбор между принципиально разными вариантами.

## 4. Правило по изображениям

Картинки добавлять можно только через безопасный процесс:

1. Сначала искать реальные изображения.
2. Не использовать нестабильные hotlink-ссылки.
3. Не вставлять страницы Unsplash/Pexels/Pixabay как `src`.
4. Готовить локальные assets, например:
   - `assets/food/syrniki.jpg`
   - `assets/food/chicken-soup.jpg`
5. Генерацию использовать только после отдельного прямого разрешения Лизы.

## 5. Текущая стабильная версия

Текущая стабильная версия:

`webstable32-clean-stable-build`

Её не ломать. Новые изменения готовить аккуратно и, по возможности, сначала как документы/план или отдельный слой.

## 6. Next action

Продолжить v33 без дополнительных вопросов:

1. Переписать первую партию из 10 рецептов художественно.
2. Подготовить фото-план для первой партии.
3. Не генерировать изображения.
4. Не внедрять в код без необходимости.
5. Показывать результат готовым пакетом.
